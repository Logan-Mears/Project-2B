# Functionality
This project searches through incredibly large data sets of personal details (name, email, job, etc.) to find the details of a specific person. The term that is searched for specifically is the unique userID, as it is possible for most other data fields (aside from phone number and email address) to be the same for different people. This search is performed in two ways: hash tables and tries.

# Purpose
This project is useful as it compares the efficiency of hash tables vs. tries when attempting to search differing sizes of databases, giving a practical demonstration of the sizes at which each data structure is most useful while also being able to be used in the real world in a variety of ways, such as customer/employee information at businesses, health records, etc.
